  1. <b>Setting up a product</b><br>
  2. Before you can report issues, you need to have at least one product. Or project. In BUGS 2, we call the top level a <b>product</b>. Doesn't mean 
  3. you cannot use BUGS 2 for projects, and other things - but it's nice to know what we're talking about.<br>
  4. <br>
  5. So, you need a product. If you're not there already, click the <span style="display: float;"><img src="themes/oxygen/cfg_icon_products.png">&nbsp;<b>Manage products</b></span> link.
  6. Here, you have a list of all the products in BUGS 2 (that you have access to), and you can create new products. You can also set which product should be 
  7. the default product for users when they report new issues.<br>
  8. <br>
  9. To create a new product, enter the product or project name in the <b>"ADD A PRODUCT"</b>-field, then click <b>Add</b>-button with the plus-sign next 
  10. to it. That's all you need to create a product!<br>
  11. <br>
  12. <b>IMPORTANT: Remember to give other users access to the product.</b><br>Access will be given to the group you are in, but remember to grant access to
  13. other groups or users as well.<br>
  14. <br>
  15. Of course, that's not all the can be done with produts in BUGS 2. Let's look at the different settings. Click the <br>
  16. <span style="display: float;"><img src="themes/oxygen/cfg_icon_productsettings.png">&nbsp;<b>Information &amp; settings</b></span> link.<br>
  17. <br>
  18. <b>Information &amp; settings</b><br>
  19. The first four input fields is for the name, description, homepage and documentation homepage for your product:
  20. <ul>
  21.     <li><b>Description</b><br>Will appear on the overview page for your product, which will be available from the frontpage</li>
  22.     <li><b>Homepage</b><br>Will appear as a link on the overview page (see <i>description</i>) and in the product block on the frontpage</li>
  23.     <li><b>Documentation</b><br>Will appear as a "View documentation"-link on the product overview page, and on the frontpage</li>
  24. </ul>
  25. You also have a setting for "prefix". You should use prefixes - they make your life easier. More information about using a product prefix is found
  26. here: <a href="?topic=prefix&prev_topic=setup_product"><b>What is a prefix?</b></a><br>
  27. <br>
  28. There are also several product settings:
  29. <ul>
  30.     <li><b>Lead by</b><br>Who's in charge of this product. Can be a team or a user. The user/users in the team will be notified of new issues, and the
  31.     user/team will appear on the frontpage in the info block for this product.</li>
  32.     <li><b>QA by</b><br>Same as above, only for QA</li>
  33. </ul>
  34. You can also assign developers to different part of your product - more about that in the next section (see "setting up editions and components")<br>
  35. <ul>
  36.     <li><b>Use tasks in issue reports</b><br>If "yes", you can add tasks to issue reports. Tasks can split the issue in to smaller parts, and can be
  37.     tracked individually. Tasks have statuses and assignees, and they can also be promoted to separate issues if they turn out to be that</li>
  38.     <li><b>Allow voting for issues</b><br>Lets people post votes on issues, letting developers know which issues to prioritise</li>
  39.     <li><b>Time measuring</b><br>Setting for how time should be measured on issues. Issues can have "estimated" and "elapsed" time.</li>
  40.     <li><b>Hours per day</b><br>Used in conjunction with the previous setting when this is set to something including days - how many hours should be
  41.     included in each day</li>
  42.     <li><b>Release date</b><br>Select if there is a planned release date, and when it is. Will be displayed in the top of the product overview</li>
  43.     <li><b>Released</b><br>Whether the product is released or not. Displayed same as above</li> 
  44.     <li><b>Allow issues to be reported</b><br>Does what it says</li>
  45.     <li><b>Use builds</b><br>Whether or not to use builds</li>
  46.     <li><b>Default status for new issues</b><br>Which status you want new issues to have by default</li>
  47. </ul>
  48. <br>
  49. <br>
  50. Continue to <a href="help.php?topic=setup_editions&prev_topic=config_products"><b>Setting up editions and selecting components&nbsp;&gt;&gt;</b></a><br>
