  1. <div style="padding: 2px;"><b>What is a "build"?</b><br>
  2. As the product evolves, newer "builds" may be released. A "build" is a release at a given time, and each "build" is 
  3. identified by a corresponding "version number".<br>
  4. <br>
  5. You may have seen this in software you are using - version numbers are such as "1.1", "2.1.3", etc. This information
  6. is usually available in the programs "about"-menu.<br>
  7. <br>
  8. <b>Ex.:</b> If you click the "About"-link on this page, you can see which "build" this program has. (If you click it, 
  9. you can click the "Continue reporting" link to get back to the reporting page)<br>  
  10. <br>
  11. <b>What build do I select?</b><br>
  12. Try to select the same build number as the application you are using. Use the information on this page to find out how
  13. to find the build number. If you are unsure, but you know you have the latest version, just use the default.<br>
  14. <br>
  15. <b>The build I want to report an issue for is not on the list</b><br>
  16. In this case, it might not be possible to report an issue for that build. If so, try contacting the developers, either
  17. by registering on the page and sending them a message, or look for a "contact us" link.<br>
  18. <br>
