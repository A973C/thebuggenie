  1. <div style="padding: 2px;"><b>Was ist eine Dringlichkeit?</b><br>
  2. Es existieren normalerweise verschiedene "Dringlichkeits-Levels" welche den Entwicklern mitteilen, wie erst/kritisch 
  3. Ihr Sachverhalt ist. 
  4. There are usually different "severity levels" which tells the developers how serious/critical your issue is. 
  5. Sie sollten bei der Auswahl des Dringlichkeits-Levels eher zurückhaltend agieren. Das bedeutet, dass ein in Ihren Augen 
  6. dringender Sachverhalt nicht unbedingt generell dringend ist.
  7. <br>
  8. <br>
  9. <b>Woher weiß ich, welches "Dringlichkeits-Level" ich wähle?</b><br>
  10. Diese haben normalerweise selbsterklärende Namen wie zum Beispiel "Kritisch", "Gering", "Normal", usw. Schauen Sie nach
  11. einem Namen, den Sie für passend halten.
  12. <br>
  13. <br>
  14. <i>Sie sollten daran denken, dass "Kritisch" normalerweiße für Sachverhalte reserviert ist, die die generelle 
  15. Funktionsweise der Software beeinflussen.</i><br>
  16. <br> 
  17. <u>Falls Sie nicht das passende "Dringlichkeits-Level" finden,</u> wählen Sie "Unbekannt" oder "Gering" - 
  18. diese existieren normalerweise.
