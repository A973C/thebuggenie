  1. Si se especifica un prefijo, cada problema informado para un producto tendrá el prefijo delante del número de problema.<br>
  2. <br>
  3. Se puede dejar vacío, pero esto no es conveniente, se debe mantener abreviado.<br>
  4. <br>
  5. <b>Por ejemplo</b><br>
  6. Supongamos que tiene 3 productos, <i>Producto A</i>, <i>Producto B</i> y <i>Producto C</i>. Usted entonces quiere un prefijo para sus productos:<br>
  7. <ul>
  8. <li>Producto A tiene el prefijo <b>PA</b></li>
  9. <li>Producto B tiene el prefijo <b>PB</b></li>
  10. <li>Producto C tiene el prefijo <b>PC</b></li>
  11. </ul>
  12. Cuando el primer problema sea informado en el <i>Producto A</i>, el número de problema que tendrá será <b>PA-1</b>. El próximo problema será llamado <b>PA-2</b>. El primer problema informado en el <i>Producto B</i> se llamará por supuesto <b>PB-1</b>, y el próximo será numerado correctamente - <b>PB-2</b>.<br>
  13. <br>
  14. Al usar un prefijo en sus productos, puede fácilmente ver - sólamente al mirar el número - que producto es el afectado.<br>
  15. <br>
  16. Si no quiere utilizar un prefijo, cada problema será numerado secuencialmente. El primer problema informado será el  <b>#1</b>, el próximo <b>#2</b> - sin identificar el producto afectado. Por supuesto que se puede tener un producto con prefijo, y otro sin, combínelos si prefiere.
