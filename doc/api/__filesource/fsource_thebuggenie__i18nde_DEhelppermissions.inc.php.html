  1. Im Gegensatz zu BUGS besitzt BUGS 2 ein sehr leistungsstarkes Zugriffsrechte-System. Einfach zu verstehen und zu bedienen, jedoch sehr flexibel.<br>
  2. <br>
  3. Die Zugriffsrechte in BUGS 2 basieren auf 4 Stufen (sortiert nach Wichtigkeit):
  4. <ul>
  5.         <li>Für einen bestimmten Benutzer</li>
  6.         <li>Für Mitglieder eines Teams</li>
  7.         <li>Für Mitglieder einer Gruppe</li>
  8.         <li>Für jeden</li>
  9. </ul>
  10. Die Zugriffsrechte sind zuerst definiert für "Jeder", dann eine Gruppe, dann ein Team, und dann für einen bestimmten Benutzer. Normalerweise sind die ersten beiden genug. Wie auch immer, es ist wichtig zu wissen in welcher Reihenfolge die Zugriffsrechte angewendet werden:
  11. <ul>
  12.         <li>Benutzer-Rechte überscheiben alles, sofern es benutzerspezifisch ist</li>
  13.         <li>Team-Rechte überschreiben Gruppen-Rechte</li>
  14.         <li>Gruppen-Rechte überschreiben "Jeder"-Rechte</li>
  15.         <li>Das "Jeder"-Recht ist das Standard-Recht und wird von allen überschrieben</li>
  16. </ul>
  17. Wenn Sie die Zugriffsrechte festlegen, erscheinen farbige Symbole, die Ihnen zeigen, welche Zugriffsrechte für das angegebene Objekt erteilt wurden:
  18. <div style="padding: 10px; margin-left: 15px;">
  19.         <p><img src="themes/default/led_green.png"> - Voller Zugriff auf Objekt. Für Objekte mit Lese- / Schreibzugriff bedeutet grün voller Lese- / Schreibzugriff.</p>
  20.         <p><img src="themes/default/led_yellow.png"> - Begrenzter Zugriff auf angegebenes Objekt - bei Objekten mit Lese- / Schreibzugriff haben Sie nur Lesezugriff.</p>
  21.         <p><img src="themes/default/led_red.png"> - Kein Zugriff auf angegebenes Objekt.</p>
  22.         <p><img src="themes/default/led_lightblue.png"> - Keine Zugriffsrechte für Benutzer/Team/Gruppe erteilt. Wenn ein blaues Symbol erscheint, wurde der Zugriff über eine untere Stufe erteilt.</p>
  23. </div>
  24. Wenn Sie festlegen wollen, was "Jeder" machen darf, können Sie das unter <b>Einstellungen &ndash;&gt; Teams &amp; Gruppen verwalten</b>.
  25. Die Zugriffsrechte, die in der Gruppe "Jeder" erteilt werden, gibt an, was ein Benutzer darf oder worauf er Zugriff hat. Nachdem Sie festgelegt haben was "Jeder" darf,
  26. können Sie festlegen was Benutzer der verschiedenen Gruppen dürfen. Standardmäßig existieren eine "Administrator"-Gruppe, eine "Gast"-Gruppe und eine "Benutzer"-Gruppe.
  27. Wählen Sie eine dieser Gruppen aus um die Zugriffsrechte für die Benutzer ein dieser Gruppe festzulegen.<br>
  28. <br>
  29. Individuelle Benutzer-Rechte werden über <b>Einstellungen &ndash;&gt; Benutzer verwalten</b> festgelegt.
  30. Suche Sie einen Benutzer und klicken Sie auf "Berechtigungen setzen" bei diesem Benutzer.<br>
  31. <br>
  32. <b>Jetzt schauen wir uns das mal an einem Beispiel an</b><br>
  33. Benutzer in der "Administrator"-Gruppe soll Zugriff zu den "Einstellungen" haben. Wählen Sie die "Administrator" Gruppe aus und klicken Sie
  34. auf das blaue "Einstellungen" Symbol. Dies berechtigt allen Benutzern in der "Administrator"-Gruppe den Zugriff auf den "Einstellungen" Verweis
  35. im oberen Menü, sowie auf die Einstellungen-Seite.<br>
  36. <br>
  37. Jetzt geben wir den verschiedenen Modulen unter den "Einstellungen", durch das Klicken auf "Einstellungen", die Zugriffsrechte.
  38. Jetzt Klicken Sie auf die blauen Symbole für jedes Modul, dem Sie <i>Lesezugriff</i> geben wollen.
  39. Durch nochmaliges Anklicken des (jetzt gelben) Symbols erteilen Sie diesem Modul ebenfalls <i>Schreibzugriff</i>.
  40. Durch weiteres Anklicken des (jetzt grünen) Symbols verweigert diesem Benutzer Zugriff zu diesem Modul (das ist normalerweise nur nötig, wenn
  41. der Zugriff bereits anhand einer unteren Stufe erteilt wurde). Erledigen Sie dies für die Module "Benutzer verwalten" sowie für "BUGS 2 Bereiche".
  42. <br><br>
  43. Nehmen wir an, dass Sie die Benutzer- und Bereichsverwaltung nur für "Mitarbeiter" zugänglich machen wollen. Stellen Sie sicher, dass Sie
  44. den Zugriff auf "Einstellungen" -> "Benutzer verwalten" keiner Gruppe (inkl. der "Jeder" Gruppe) oder keinem Benutzer gewährt haben.
  45. Jetzt erstellen Sie ein <i>Team</i> "Mitarbeiter" und wählen es durch Anklicken aus. Jetzt wählen Sie das Modul "Einstellungen"
  46. durch Anklicken aus (alle Symbole sollten blau sein). Jetzt klicken Sie auf doppelt auf das "Benutzer verwalten" Zugriffs-Symbol, damit es grün wird.
  47. Führen Sie die selbe Aktion für "BUGS 2 Bereiche" aus.<br>
  48. <br>
  49. <b>Glückwunsch!</b><br>
  50. Jetzt haben alle Benutzer in der Administrator-Gruppe Zugriff auf alle Module unter Einstellungen, außer "Benutzer verwalten" und "BUGS 2 Bereiche",
  51. welche nur für Benutzer, die Mitglieder des Team "Mitarbeiter" sind, zugänglich sind.<br>
  52. <br>
  53. Bedenken Sie, dass Benutzer nur einer Gruppe angehören können, jedoch verschiedenen Teams angehören können. Denken Sie ebenfalls daran, dass
  54. die Einstellung Zugriff "verweigern" nur gültig ist, solange nicht Zugriff "erlauben" auf der selben Ebene oder einer Ebene höher gewählt wurde.
  55. Mehr über die Stufen des Zugriffs sowie über das Überschreiben von Zugriffsrechten erfahren Sie am Anfang dieses Hilfethemas.
