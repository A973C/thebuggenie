  1. <b>Setting up SVN integration</b><br>
  2. To set up SVN integration, follow these three steps:
  3. <ul>
  4.     <li><b>Add post-commit hook to SVN repository</b><br>
  5.     In the module directory, there is a post-commit.sh script that needs to be triggered upon an SVN commit. If you don't use any other commit-hooks, this script can replace the default post-commit script. If you already have a working post-commit.sh script, copy the contents of that file into your existing post-commit.sh script.<br>
  6.     <b>Remember to remove the post-commit.sh from the module directory, or make it inaccessible from the web</b><br>
  7.     <br>
  8.     If you don't know how to set this up, please consult the SVN manual.<br><br></li>
  9.     <li><b>Edit post-commit script</b><br>
  10.     Make sure you edit the script so the correct path is used. If you are going to use the web-update, make sure the settings in the file matches the settings on the SVN integration configuration page.<br><br></li>
  11.     <li><b>Set up ViewVC</b><br>
  12.     ViewVC is a web interface for browsing repositories. To use this, please set it up according to the instructions on the <a href="http://www.viewvc.org" target="_blank">ViewVC website</a>.<br>
  13.     <br>
  14.     When ViewVC is set up, make sure you set up the URLs to ViewVC in the SVN integration configuration panel.
  15. </ul>
  16. <br>
  17. When you have followed the three steps above, you should be able to use the SVN integration module.<br>
  18. <br>
  19. To learn more about how to use the SVN integration properly, please see <a href="help.php?topic=svn_integration/main"><b>Using SVN integration</b></a>.
