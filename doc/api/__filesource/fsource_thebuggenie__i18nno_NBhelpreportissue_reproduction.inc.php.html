  1. <div style="padding: 2px;"><b>What should "reproduction steps" contain?</b><br>
  2. This field is usually optional. Filling out this field helps developers identify your problem, by being able to reproduce it. If you put in details here
  3. about how you managed to "trigger" the issue you are reporting, developers can find and identify the issue easier.<br>
  4. <br>
  5. Remember that you can use the formatting toolbar below the text field to help you describe the reproduction steps.<br>
  6. <br>
  7. <b>Do you have some examples of <u>good</u> "reproduction steps"?</b>
  8. <ul>
  9.     <li>When I click the <u>F</u>ile-&gt;<u>O</u>pen menu item, then double-click a PDF in the browser that appears, Program X crashes with an error message</li>
  10.     <li>When I press the "print" button in the menu bar (the one with the printer icon), an error message pops up</li>
  11.     <li>Open program X, click "help", then search for "candy". Open the second item in the list ("candy bars"), then notice the spelling error in the header for the second paragraph</li>
  12. </ul>
  13. The same thing goes for reproduction steps as for issue descriptions - they can never be <i>too</i> thorough.<br>
  14. <br>
