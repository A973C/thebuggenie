Documentation is available at prefix.inc.php

  1. If you specify a prefix for one of your products, every issue reported for that product will have that prefix in front of the issue number.<br>
  2. <br>
  3. You can use anything as a prefix, but for your own convenience, you should keep it short.<br>
  4. <br>
  5. <b>An example</b><br>
  6. Say you have three products, <i>Product A</i>, <i>Product B</i> and <i>Product C</i>. You would then want to prefix your products like this:<br>
  7. <ul>
  8. <li>Product A has prefix <b>PA</b></li>
  9. <li>Product B has prefix <b>PB</b></li>
  10. <li>Product C has prefix <b>PC</b></li>
  11. </ul>
  12. When the first issue is reported for <i>Product A</i>, the issue number for that issue will then be <b>PA-1</b>. The next issue will be called <b>PA-2</b>. The first issue reported for <i>Product B</i> will of course be named <b>PB-1</b>, and the next will be numbered accordingly - <b>PB-2</b>.<br>
  13. <br>
  14. By using a prefix for your products, you can then easily see - just by looking at the issue number - what product that issue affects.<br>
  15. <br>
  16. If you don't want to use a prefix, every issue will be numbered sequentially. The first issue reported will be numbered <b>#1</b>, the next will be numbered <b>#2</b> - regardless of which product it affects. You can of course have one product with prefix, and one without - mix and match as you please.
